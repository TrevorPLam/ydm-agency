## Purpose
Guide web performance implementation with Core Web Vitals tracking, custom metrics, performance reporting, and optimization recommendations.

## When to Invoke
- When implementing web vitals packages
- When setting up performance monitoring
- When optimizing page speed

## Prerequisites
- Next.js configured
- Analytics configured

## Implementation Steps

### 1. Set up web vitals tracking
Use the template in `templates/web-vitals.tsx`

### 2. Add custom metrics
Reference `templates/custom-metrics.ts`

### 3. Implement performance reporting
Use `templates/performance-reporting.ts`

### 4. Add optimization recommendations
Use `templates/optimization-recommendations.ts`

## Templates
- `templates/web-vitals.tsx` - Web vitals tracking
- `templates/custom-metrics.ts` - Custom metrics
- `templates/performance-reporting.ts` - Performance reporting
- `templates/optimization-recommendations.ts` - Optimization recommendations

## Patterns
- Use web-vitals library
- Track Core Web Vitals
- Monitor performance trends

## Anti-Patterns
- Don't ignore performance budgets
- Don't skip optimization
- Don't track everything
