## Purpose
Guide Test-Driven Development with test-first methodology, red-green-refactor cycle, and testing patterns.

## When to Invoke
- When implementing TDD methodology
- When setting up test-first development
- When practicing red-green-refactor cycle

## Prerequisites
- Vitest configured
- Testing patterns established

## Implementation Steps

### 1. Write failing test
Start with a failing test that describes desired behavior

### 2. Implement minimal code
Write minimal code to make test pass

### 3. Refactor
Refactor code while keeping tests green

### 4. Repeat
Continue cycle for all features

## Templates
- `templates/test-pattern.ts` - Test pattern template
- `templates/tdd-cycle.md` - TDD cycle guide

## Patterns
- Write test before implementation
- Keep tests small and focused
- Refactor confidently with test coverage

## Anti-Patterns
- Don't write tests after code
- Don't skip the refactor step
- Don't write integration tests as unit tests
